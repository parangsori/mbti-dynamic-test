# DAY4 Working Entry

이 저장소에서 새 채팅을 시작하면, 다른 분석보다 먼저 아래 순서로 진행합니다.

1. `PROJECT_PRINCIPLES.md`를 먼저 읽고 현재 원칙, 로드맵, 최근 이슈를 파악한다.
2. 이번 요청과 직접 관련된 현재 변경 파일을 확인한다.
3. 관련 `backups/` 폴더와 이전 정상 구조를 비교할 필요가 있는지 먼저 판단한다.
4. 필요한 최소 테스트만 선택하고, 테스트가 길어질 가능성이 있으면 먼저 사용자에게 알린다.

핵심 강제 원칙:

- 새 채팅에서는 기존 맥락을 추정하지 말고 원칙 문서부터 확인한다.
- 사소한 변경 포함 모든 repo 변경 전 작업 단위 백업을 먼저 만든다.
- 요청받은 범위만 수정하고, 기존 정상 동작 영역에 회귀를 만들지 않는다.
- 문제가 반복되면 현재 코드만 보지 말고 백업본과 diff 비교를 우선한다.
- 자동 검증은 짧고 효과적인 범위로 유지하고, 시간이 길어지는 UI 확인은 사용자 빠른 테스트로 전환할 수 있다.

상세 운영 기준, 버전 반영 기준, 현재 우선순위는 `PROJECT_PRINCIPLES.md`를 기준으로 따른다.
